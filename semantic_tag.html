HTML5 문서

HTML5에는 태그의 이름만 보고서도 문서 구조에서 어떤 역할을 하는지 쉽게 이해할 수 있는 '시멘틱 태그' 가 포함되었다.

문서 구조를 위한 HTML5 시멘틱 태그

header 태그 - 머리말

특정 부분의 머리말에 해당. html안의 head와는 다르다. 주로 페이지 맨 위쪽이나 왼쪽에 삽입하며 헤더의 내용으로는 주로 form태그를 사용해 검색창을 넣거나, nav태그를 사용해 사이트 메뉴를 넣는다.

nav 태그 - 문서를 연결하는 네비게이션 링크

쉽게 말해 링크 모음? 메뉴 모음 정도로 이해. 다른 문서로 이동할 수 있도록. 헤더, 푸터, aside 등에 포함 시킬 수도, 독립적으로 사용할 수도 있다.

header+nav 테스트

확인하기

section 태그 - 주제별 콘텐츠 영역 나타내기

종류별로 나눌때 사용. 그리고 h1~h6 제목을 이용하여 섹션별 제목을 나타냄. section안에 section을 넣을 수 있겠지 물론?

article 태그 - 콘텐츠 내용 넣기

section과는 다르다. 기사의 의미가 있어서 실제 내용을 나타낼 때 article을 사용하는 것이 알맞다. 블로그의 포스트나 웹 사이트의 내용, 코멘트, 독립적인 웹 콘텐츠 항목 등. 다시 말해, 떼어 놓아도 완전히 하나의 콘텐츠가 된다면 article 사용하자.

aside 태그 - 본문 이외의 내용 표시

사이드바. 왼쪽, 오른쪽, 하단 등 사이드바 활용. 광고나 태그 안내 등?
